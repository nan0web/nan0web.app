# 📡 Архітектура NaN0Web CLI (v1.0)
## Оболонка One Logic — Many UI (OLMUI)

NaN0Web CLI є реалізацією універсального протоколу OLMUI. Він розглядає весь CLI-додаток як **Доменну Модель**, яка генерує **Наміри** (Intents), що рендериться за допомогою термінального **InputAdapter**.

### 1. Ієрархія та Потік
1. **Раннер (`bin/nan0web`)**: Тонкий Node.js міст для глобального виклику.
2. **Головний модуль (`src/cli.js`)**: Ініціалізує додаток та впорскує інфраструктуру.
3. **ShellModel (`src/domain/ui/cli/ShellModel.js`)**: "Мозок" CLI. Визначає команди як частину своєї статичної схеми.
4. **InputAdapter (`@nan0web/ui-cli`)**: Універсальний інтерфейс для `prompt`, `log`, `ask` та `progress`.

### 2. Основні Паттерни
- **Late-Bound DI**: Інфраструктура (AppRunner, DB, spawn) впорскується в ShellModel під час створення екземпляра.
- **Позиційне Відображення**: Аргументи CLI зіставляються зі статичним полем `command` ShellModel за допомогою `resolvePositionalArgs`.
- **Чисті Гененатори**: Усі побічні ефекти (логування, інтерактивні форми) виконуються виключно через `yield log(...)` або `yield ask(...)`.

### 3. Виклик Локального Раннера (Міст)
Коли `nan0web cli` викликається всередині конкретного проекту:
1. `ShellModel` розпізнає `command = 'cli'`.
2. Він шукає локальний раннер `src/ui/cli/index.js` у поточній папці (`cwd`).
3. Він запускає `node`, передаючи абсолютний шлях до локального раннера.
4. Він прокидає всі прапорці (`--dsn`, `--locale`) та додаткові позиційні аргументи в дочірній процес.

### 4. Протокол Верифікації
- Усі генератори ShellModel мають бути протестовані через `runGenerator` з моковим адаптером.
- Статичні поля повинні використовувати `Object.getOwnPropertyNames` для коректного виявлення метаданих (MaSaA v2).
- Локальні раннери повинні реалізувати перевірку `isMain` за допомогою `path.resolve(process.argv[1])` для підтримки глобального виклику.
