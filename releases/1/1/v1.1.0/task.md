# Місія релізу v1.1.0

## Scope (Що саме робимо)

Інтеграція системи команд у ядро `nan0web.app`, створення мінімального істинного набору даних для перевірок.
Додаток має працювати як "Data-Driven Runner" на базі файлової структури, без `pages.yaml`.

### 1. Тестові Дані (The Baseline Data)
Створюємо структуру директорій, яка є джерелом правди. Використовуємо формат `nan0` для перевірки його повноцінної роботи. Маршрутизація автоматична (In-Memory File Routing), а для меню (навігації) використовується `nav.nan0` окремо для кожної мови (оскільки мова несе в собі код мислення, пріоритети пунктів можуть відрізнятися):
```text
data/
├── _/
│   ├── t.nan0          # англійський базовий словник (генерація з ключів)
│   └── nav.nan0        # навігація для англійського мислення
├── catalog/
│   └── index.nan0      # list layout ($content)
├── contact/
│   └── index.nan0      # form layout ($content)
├── uk/
│   ├── _/
│   │   ├── t.nan0      # український словник
│   │   └── nav.nan0    # навігація для українського мислення
│   ├── catalog/
│   │   └── index.nan0  # uk-mirror: list layout
│   ├── contact/
│   │   └── index.nan0  # uk-mirror: form layout
│   └── index.nan0      # entry point (uk)
└── index.nan0          # entry point (en)
```
У `nav.nan0` ми зберігаємо лише масив посилань (href, title як i18n key, icon). Меню автоматично читатиметься `AppRunner`-ом і передаватиметься адаптерам.

### 2. Сценарії Користувача (User Stories)
Відмова від звичайних обривчастих тестів. Впроваджуємо `test:stories` — канонічні сценарії використання:
- **Story 1 (CLI Proxy)**: Перевірка проксування команди `nan0web cli` до `nan0cli`.
- **Story 2 (Build & Pages)**: Перевірка збірки `nan0web build` та файлового роутингу (конвертація дзеркальних директорій `[locale]/[route]/index.nan0` у dist JSON).
- **Story 3 (Dev & API)**: Запуск `dev` та API.
- **Story 4 (Galleries)**: Формування галерей.
- **Story 5 (Nan0 vs YAML Benchmark)**: Написання тесту продуктивності (бенчмарку) для порівняння швидкості парсингу `YAML` проти `NAN0`.
Кожна Story описується в окремому `*.story.js` (як TDD Contract).

### 3. Оновлення CLI Adapter (`src/runner.js` та `src/cli.js`)
- `nan0web cli` та подальші виклики прокидаються безпосередньо у `nan0cli` із `@nan0web/ui-cli`.
- Роутинг повністю делегується парсингу файлів. Видалити `PagesRouter` як зайву прив'язку, використовувати архітектуру Dynamic Hub Routing.

## Definition of Done (Acceptance Criteria)

- Створені тестові дані (`data/en/` та `data/uk/`) у відриві від хардкоду у форматі `nan0`.
- Переписано `src/cli.js` для виклику `nan0cli`.
- Усі `*.story.js` тести написані і проходять (зокрема підтверджено суттєву перевагу `.nan0` у продуктивності над `.yaml`).
- `test:stories` інтегровано в цикл тестування (`test:all`).
