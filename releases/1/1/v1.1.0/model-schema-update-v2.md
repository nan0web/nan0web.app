# Доповнення до 0HCnAI Model-as-Schema Workflow (v1.1.1)

### 🌳 Ієрархічне моделювання (`children`)
Для опису деревоподібних структур (меню, карти сайту, ієрархії команд) використовуй поле `children`, щоб відповідати HTML конвенціям.

```javascript
static children = {
    help: 'Sub-items',
    type: 'MyModel[]',
    hint: MyModel, // Пряме посилання на клас моделі для рекурсії
    default: []
}
```

### 🎯 Використання `hint`
Параметр `hint` у статичних метаданих вказує на клас моделі. Це дозволяє:
1. **Генератору схем**: Рекурсивно знаходити та генерувати схеми для вкладених об'єктів.
2. **Адаптерам**: Автоматично перетворювати вкладені об'єкти (чи масиви) на екземпляри відповідних класів моделей під час ініціалізації (напр. через `resolveDefaults` чи конструктор).

### 🏷 Аліаси та Резолвінг
- `alias: 'name'` — використовується для мапінгу "зручного" імені поля на "істинне" ім'я у форматі даних (напр. у YAML/JSON).
- `resolveAliases(data)` — функція (має бути реалізована), яка готує дані перед ініціалізацією об'єкта, перетворюючи аліаси на робочі змінні.
